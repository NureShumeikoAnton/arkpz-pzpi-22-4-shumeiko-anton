ДОДАТОК А
VISION AND SCOPE DOCUMENT (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)

1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1 Background (Передумови)

Сучасні системи клімат-контролю відіграють важливу роль у забезпеченні комфорту та енергоефективності в приміщеннях, проте більшість таких рішень є універсальними та не враховують індивідуальних потреб користувачів. Для багатьох споживачів стає проблемою адаптація стандартних налаштувань до специфіки їхнього повсякденного життя, що призводить до зниження зручності, підвищеного споживання енергії та дискомфорту.
Додатковою проблемою є відсутність простого механізму інтеграції особистих налаштувань у кліматичну систему, їх збереження та швидкої зміни профілів, зокрема з урахуванням таких факторів, як графік перебування в приміщенні, особисті вподобання щодо температури та вологості, а також екологічні параметри.
Таким чином, необхідно розробити програмну систему клімат-контролю з підтримкою персоналізованих профілів користувачів, яка зможе автоматично адаптувати параметри мікроклімату залежно від їхніх індивідуальних потреб. Це дозволить не лише підвищити рівень комфорту, але й зменшити споживання енергії через оптимізацію роботи системи.

1.2 Business Opportunity (Бізнес-можливості)

Ринок систем клімат-контролю стрімко розвивається завдяки попиту на автоматизацію домашніх процесів і підвищення енергоефективності. Більшість існуючих рішень пропонують базові функції налаштування температури, вологості та інших параметрів, проте їхні можливості часто обмежені відсутністю персоналізації та зручного управління профілями.
Одним з прикладів такого продукту є Nest Thermostat від Google. Пристрій пропонує адаптивний клімат-контроль на основі поведінки користувачів і аналізу даних. Однак, система не підтримує функцію швидкого перемикання між профілями для різних членів сім’ї або сценаріїв.
Ecobee SmartThermostat також має власне програмне забезпечення для регулювання мікроклімату, що додає функціонал голосового управління та геолокації. Це дозволяє регулювати температуру на основі присутності користувачів у будинку. Проте система більше фокусується на індивідуальній оптимізації, ніж на створенні декількох профілів, що легко зберігаються та перемикаються.
Отже, мета запропонованої системи полягає у наданні можливості швидкого створення, збереження та перемикання особистих профілів користувачів. що є основною проблему серед перелічений аналогів. Система буде включати у себе комплекс IoT-датчиків, що буду контролювати клімат у приміщенні та адаптувати його під активні налаштування користувача.

2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

	2.1 Vision Statement (Окреслення концепції)

	Метою програмної системи клімат-контролю з особистими профілями є створення інтелектуального та персоналізованого середовища для автоматизації управління мікрокліматом у приміщеннях. Система забезпечує адаптацію до індивідуальних потреб користувачів, підвищуючи комфорт та енергоефективність.
	Для збереження профілів з використанням Rest API користувач зможе використовувати інтерфейс веб-версії додатку, а для швидкої зміни профіля зможе скористатися мобільним пристроєм. Також за допомогою мобільного застосунку користувач зможе переглянути поточні показники IoT-датчиків, які система буде використовувати для збору даних про температуру, вологість, швидкість вентиляції та інші параметри в реальному часі. Ці дані дозволяють автоматично налаштовувати оптимальні умови для користувачів, які вони задали. Також система буде займатись формуванням статистики для користувача, виділяючи найнапруженіші періоди для системи клімат-контролю, вираховуючи приблизне споживання енергії за певними вихідними даними. Також фіксуються моменти, при яких обладнання не може досягти поставленої цілі.
	Отже, програмна система для клімат-контролю з особистими профілями дозволить користувачам швидко маніпулювати мікрокліматом приміщення, забезпечуючи комфортні умови та зберігаючи електроенергію. Наявність збережених особистих профілів у свою чергу дозволить користувачам не турбуватися про втрату комфортних налаштувань, а також створити декілька програм для різних сезонів чи задач.

	2.2 Major Features (Головна функціональність)

MF-1. Аналіз даних IoT-сенсорів: використання датчиків температури, вологості, та якості повітря для автоматичного регулювання параметрів кліматичного обладнання для адаптації клімату відповідно до активного профілю;
MF-2. Швидка зміна профіля: надання користувачу зручного інтерфейсу мобільного додатку для вибору активного профілю та моніторингу поточних показників сенсорів;
MF-3. Персональне налаштування клімат-контролю: надання можливості для створення користувачем декількох профілів налаштувань клімат-контролю через веб-застосунок. Можливість їх збереження, видалення та редагування;
MF4. Формування статистики показників: система зберігає отримані дані та автоматично аналізує параметри, такі як температура, вологість і швидкість вентиляції, формуючи графіки, виділяючи граничні значення та медіани. Також система автоматично проінформує користувачів у випадках, коли обладнання не може скорегувати клімат у приміщенні;
MF5. Система сповіщень: надсилання автоматичних повідомлень користувачам у разі виникнення критичних змін кліматичних умов або перевищення встановлених меж;
MF6. Енергозберігаючі алгоритми: впровадження механізмів оптимізації енергоспоживання через автоматичне налаштування режимів роботи обладнання без шкоди для комфорту користувачів;
MF7. Робота із таймером: система дозволяє користувачу, використовуючи мобільний застосунок, задати таймер на вимкнення, якщо він, наприклад, покидає домівку на певний непередбачений проміжок часу.

3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)

	3.1 Scope of Initial Release (Рамки первинного випуску)

Початкова версія системи буде зосереджена на впровадженні базового функціоналу серверної системи для IoT-пристроїв, який забезпечить базу для подальшої розробки користувацьких інтерфейсів для веб- та мобільного застосунку.
Функції IoT-пристроїв:
1. Моніторинг параметрів навколишнього середовища: постійний збір даних із підключених сенсорів для відправлення на сервер;
2. Реакція на команди сервера: впровадження можливості отримання команд від сервера для зміни режиму роботи пристроїв (наприклад, увімкнення або вимкнення вентиляції);
3. Локальна автономність: короткострокове збереження даних у разі втрати з’єднання із сервером з подальшою їх передачею при відновленні зв’язку.
Функції серверної частини:
1. Обробка даних від IoT-пристроїв: сервер отримує дані від сенсорів (температура, вологість, освітлення) у реальному часі, зберігаючи їх у базі даних для подальшого використання;
2. Управління подіями та сповіщеннями: автоматична перевірка отриманих даних на відповідність встановленим пороговим значенням із подальшим надсиланням сповіщень у разі порушення;
3. Реєстрація користувачів та управління доступом: забезпечення механізмів створення облікових записів, авторизації та надання доступу до функціоналу системи залежно від ролі користувача;
4. Зберігання історичних даних: сервер накопичує та структурує історичні дані для подальшого аналізу через веб- або мобільний клієнт;
5. Аналіз історичних даних: сервер використовує збережені дані для формування статистичних даних про середні значення мікроклімату, витрати електроенергії в певні проміжки часу, визначення пікових навантажень.
 
3.2 Scope of Subsequent Releases (Рамки наступних випусків)

Функціональність веб-застосунку:
1. Створення особистих профілів: надання можливостю користувачу створювати нові, видаляти та редагувати існуючі профілі налаштувань клімат-контролю за допомогою веб-застосунку;
2. Аналіз історичних даних: надання інструментів для перегляду історичних даних у вигляді таблиць і графіків, що допоможе користувачам аналізувати динаміку зміни параметрів зберігання;
3. Керування користувачами та ролями: функція додавання нових користувачів, призначення їм ролей (адміністратор, оператор тощо) та редагування прав доступу для забезпечення гнучкого управління системою;
4. Адміністрування пристроїв: веб-інтерфейс для віддаленого управління IoT-пристроями, включаючи їх налаштування, оновлення програмного забезпечення та реєстрацію нових пристроїв.
Функціональність мобільного застосунку:
1. Сповіщення в режимі реального часу: надання користувачам push-сповіщень про перевищення порогових значень сенсорів, критичні події або оновлення системи;
2. Доступ до параметрів середовища: можливість переглядати актуальні дані з сенсорів та отримувати короткий звіт про поточний стан у мобільному додатку;
3. Перемикання збережених профілів: забезпечення можливості перемикання особистих профілів, збережених раніше за допомогою користувацького інтерфейсу;
4. Офлайн-режим: частковий доступ до даних, завантажених раніше, у разі відсутності підключення до інтернету з подальшою синхронізацією після відновлення з’єднання.

4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)

	4.3 Operating Enviroment (Робоче середовище)

	У розробці програмної системи клімат-контролю буде використано сучасний стек технологій, спрямований на забезпечення високої продуктивності, гнучкості та зручності в роботі. Нижче описано технології, інструменти та вимоги до системи:
	Технології:
1. Java Spring: використовується для створення серверної частини додатка. Java Spring забезпечує швидкий розвиток, масштабованість, та інтеграцію з базами даних і сторонніми сервісами. Ця технологія підходить для реалізації REST API та обробки даних з IoT-сенсорів.
2. React: використовується для розробки інтерфейсу користувача. React забезпечує швидку та ефективну візуалізацію даних у реальному часі, а також надає можливість створювати інтерактивні елементи для зручного керування додатком.
3. Kotlin: використовується для створення мобільного додатка. Kotlin надає сучасний синтаксис та високу продуктивність для роботи з Android-платформою, що дозволяє легко інтегрувати мобільну версію з сервером і базою даних.
4. MySQL: реляційна база даних для зберігання даних користувачів, документів і даних з IoT-сенсорів. MySQL забезпечує високу продуктивність, надійність та підтримку складних запитів для аналізу даних.
5. Arduino: використовується для взаємодії з IoT-сенсорами. Arduino забезпечує збір даних з фізичних сенсорів і передачу їх на сервер для подальшої обробки та збереження.
Вимоги до системи:
1. Доступність: система повинна бути доступною 24/7 для користувачів, включаючи веб-додаток, мобільний додаток та інтеграцію з IoT-сенсорами. Це включає мінімальні простої та підтримку на різних пристроях в будь якому місці із наявним підключенням до мережі; 
2. Надійність: усі компоненти системи повинні працювати без збоїв, навіть за великої кількості запитів. Передбачено резервне копіювання даних і обробка можливих помилок для запобігання втраті інформації;
3. Продуктивність: система повинна швидко реагувати на запити користувачів і забезпечувати реальну взаємодію з IoT-пристроями. Серверна частина оптимізована для роботи з великими обсягами даних, а клієнтська — для швидкого відображення інформації;
4. Цілісність: всі дані повинні бути захищені від несанкціонованого доступу, а також забезпечена їх правильність і актуальність. Використання механізмів валідації та захисту гарантує високу якість оброблюваних даних.
